    FLAPPY ARDUINO

    A continuación, les mostramos uno de los juegos retro por excelencia, el Flappy Bird, adaptado para Arduino.

    ¡Disfrutad del juego!

    Descripción del proyecto

    El proyecto consiste en el juego tradicional del flappy bird, pero adaptado a Arduino. En él, tendremos que controlar el vuelo de un pájaro e intentar sobrepasar los diferentes obstáculos del camino. Cada vez que sobrepasemos uno, sumaremos un punto.

    Para ello, tendremos que mover la palma de la mano arriba y abajo sobre un sensor de distancia, y el pájaro subirá y bajará respectivamente.

    Además, tendremos otras funcionalidades como una luz led que se enciende a medida nos acercamos al sensor, un buzzer que pita cuando nos chocamos y una pantalla lcd en la que se muestran diferentes mensajes de interés.

    Autor: Noé Andreu.

    Materiales

    Los materiales que necesitaremos para implementar nuestro Pong Arduino son:

    • Arduino UNO
    • 1 Buzzer
    • 1 Pantalla lcd
    • 1 Potenciómetro
    • 1 Led, no importa el color
    • 1 Sensor HC-SR04
    • 2 Resistencias 1 KiloOhmnio

    El código

    Ahora, se muestra el código comentado que se ha implementado tanto en Processing como en Arduino para crear el juego.

    Entre ambas partes, se producirá una comunicación serial para hacer funcionar el juego de forma correcta.

    Paso a paso

    Finalmente, detallamos los pasos necesarios para la realización de nuestro Flappy Bird.

    • Primero, una vez tenemos claras las funcionalidades que queremos, hacemos un modelo digital de los componentes y las conexiones para verificar el proceso. Esto se hará con programas como Tinkercad o Fritzing, y con ello nos aseguraremos que los componentes, las conexiones o hasta el código son las correctos sin dañar los reales.
    • Montamos el circuito en la realidad. Una vez hemos verificado que las conexiones sean las correctas, montamos nuestro circuito físico.
    • Implementamos el código Arduino. Ahora, implementaremos el código Arduino, pues es el que se encarga de las funcionalidades directas de la placa.
    • Al mismo tiempo, iremos realizando el código Processing. Durante el proceso, iremos comprobando el código haciendo pruebas y solucionando las acciones una a una.
    • Probamos el juego y todas las funcionalidades implementadas. Cuando ya hayamos acabado, probamos el juego e interaccionamos de todas las formas posibles que hayamos implementado.
    • Finalmente, pulimos algunos aspectos del código o algunos errores. En nuestro caso, destacar errores con las puntuaciones (que respondían a un error en el cálculo de las posiciones) y con el sensor hc-sr04 (una mala conexión con demasiado voltaje fundió nuestro sensor y hubo que cambiarlo).
